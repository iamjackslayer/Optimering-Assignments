    Assignment Instructions

    The Simplex Method

    Important information:

    • This is the mandatory programming project in the course. You must hand in the project before the deadline in order to take the exam. In addition to that, the grading of the programming project is used at the oral examination after the course to determine your final grade. However the grade at the oral examination is mainly based on individual performance during the oral examination.
    • The programming project is to be completed and handed in by groups consisting of 1,2, or 3 members. The size of your group is left at your own discretion.
    • As with the mandatory handins: Copying the solutions of other groups constitutes cheating and is treated by ”the system” on par with exam cheating. What you hand in must entirely be your own work.


    Deadline: Friday, March 6, 12:00. This is a hard deadline!


    Programming language

    The programming project is concerned with implementing and experimenting with the simplex method in Python 3, as provided for instance by the Anaconda distribution.

    A template solution (simplex.py) is supplied which you are expected to use and follow.

    Sub-projects

    The programming project consists of 5 sub-projects which must be solved in the given order to receive points. During grading each sub-project is awarded 0, 1, or 2 points.
    Only the solution handed in before the deadline receives points.
    The minimal requirement for having the programming project obtain the status of "approved" is to solve sub-project 1 in a satisfactory way.

    Sub-project 1 (one-phase simplex method)

    • Implement the one-phase simplex method with dictionaries having entries of data type Fraction and np.float64, and using Bland's rule for pivoting.

    Sub-project 2 (experimentation)

    • Design and conduct a computational experiment evaluating the performance of your implementation for both data types Fraction and np.float64. You may for instance take inspiration from Chapter 4 of Vanderbei. Compare also your implementation using floats to scipy.optimize.linprog.

    Sub-project 3 (two-phase simplex method)

    • Extend your implementation to a two-phase simplex algorithm. You may chose either the auxiliary dictionary approach or the dual based approach.

    Sub-project 4 (pivoting rules and further experimentation)

    • Implement the largest-coefficient and the largest-increase pivoting rules. Repeat the computational experiment using these and using floats compare again to scipy.optimize.linprog.

    Sub-project 5 (integer pivoting)

    • Implement integer pivoting as described at Lecture 5 and compare experimentally against using data type Fraction. The implementation should also work for the two-phase simplex method.


    What to hand in

    Hand in the following:

    1. All your source code.
    2. A document describing and documenting computational experiments performed.


    A couple of hints

    • NumPy is built for operating on vectors and matrices in a "vectorized" way. Using this will reduce the number of required keystrokes (as well as the running time of your code!).
    • Make sure that your experiments are repeatable and usable for comparisons. If using the "random number" generator of NumPy this is ensured by setting the random seed by calling np.random.seed.
    • It may be useful to use the Python unit testing framework during implementation.
